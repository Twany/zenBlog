# Linear Working System (Core Rules)

## 1) Architecture (Chosen Model)

- Initiative = Product Line (one initiative per product line)
- Project = SOP Instance (e.g., "定位与验证" for that product line)
- Milestone = SOP Stage (e.g., 竞品与市场 / 产品定义 / SEO 先行)
- Issue = Atomic task inside a stage

## 2) Core Rules (Must Follow)

- Every new issue must belong to a product line via its Initiative and Project.
- Keep the same SOP workflow across product lines so tasks stay comparable.
- Every issue must have a milestone and a status.
- Avoid ambiguous naming: a project name must make the product line explicit.
- Do not create duplicate projects with the same name unless it is tied to a product line.

## 3) Recommended Naming Rules

- Projects:
  - Use: "<SOP Name> · <Product Line>"
- Issues:
  - Use concise verb-first titles (e.g., "收集 30 条用户抱怨")

## 4) Recommended Workflow

1. Choose product line (Initiative).
2. Choose SOP project for that product line.
3. Choose milestone (SOP stage).
4. Create issue in the correct project.
5. Set status and owner.

## 6) Optimization Suggestions

- Keep project names explicit: "<SOP Name> · <Product Line>".
- Use a single SOP template (project or issue template) and duplicate it across product lines.
- Initiatives should represent product lines; use Projects for SOP instances.
